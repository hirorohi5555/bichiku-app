# 家庭用備蓄品在庫管理システム

## プロジェクト概要

iPhoneのホーム画面に追加して使える、家庭用備蓄品の在庫管理を行うPWA（Progressive Web App）です。
緊急時でも簡単に在庫状況を確認でき、賞味期限の管理も行えるシステムです。

## 機能要件

### 備蓄品の登録
- 品名
- 数量
- 賞味期限
- カテゴリ（食品、飲料、日用品、医薬品など）

### 在庫一覧の表示
- 登録されている備蓄品の一覧表示
- カテゴリ別のフィルタリング
- 賞味期限順のソート

### 賞味期限警告機能
- 賞味期限が30日以内の商品を警告表示
- 期限切れの商品を明確に表示

### プッシュ通知機能
- Web Notification APIを使用したブラウザ通知
- 賞味期限が近づいた商品を自動で通知（30日前、7日前、当日など）
- Service Workerによる定期的なチェック
- 通知のオン/オフ設定
- 通知タイミングのカスタマイズ（ユーザー設定）

### カレンダー連携機能
- iCalendar形式（.ics）でのエクスポート機能
- 賞味期限のリマインダーをカレンダーに追加
- iPhoneの標準カレンダー/リマインダーアプリと連携
- 一括エクスポートと個別エクスポートの両対応

### 使用・補充の記録
- 備蓄品の使用時に数量を減らす
- 補充時に数量を増やす
- 履歴の記録（オプション）

### データのエクスポート/インポート
- JSON形式でのデータエクスポート
- バックアップデータのインポート
- データの引き継ぎ・共有が可能

## 開発フェーズ

### フェーズ1: MVP（Minimum Viable Product）
最小限の機能で動作する基本バージョン。以下の機能を実装：

**コア機能：**
- 備蓄品の登録（手動入力）
- 在庫一覧の表示
- カテゴリ別フィルタリング
- 賞味期限順ソート
- 賞味期限警告表示（30日以内）
- 使用・補充の記録
- データのエクスポート/インポート（JSON）

**PWA対応：**
- Service Worker（オフライン対応）
- manifest.json（ホーム画面追加）
- レスポンシブデザイン

**通知機能：**
- Web Notification APIによるプッシュ通知
- カレンダー連携（.icsエクスポート）

### フェーズ2: 入力簡易化機能
MVP完成後に追加する入力支援機能：
- バーコードスキャン機能
- テンプレート・履歴から選択
- 日付クイックボタン（3ヶ月後、6ヶ月後、1年後）
- 数量+/-ボタン
- カテゴリ自動推測

### フェーズ3: 高度な機能（将来的に検討）
- 音声入力（Web Speech API）
- OCR（賞味期限の自動読み取り）
- 統計・レポート機能
- 買い物リスト生成

## 技術スタック

### フロントエンド
- **HTML5**: セマンティックなマークアップ
- **CSS3**: レスポンシブデザイン、モダンなスタイリング
- **JavaScript**: Vanilla JS（フレームワーク不使用）

### データ保存
- **localStorage**: ブラウザローカルでのデータ永続化

### PWA対応
- **Service Worker**: オフライン対応、キャッシュ管理、定期的な賞味期限チェック
- **manifest.json**: ホーム画面への追加、アプリライクな体験
- **Web Notification API**: プッシュ通知機能
- **iCalendar形式（.ics）**: カレンダーアプリとの連携

### デザイン
- レスポンシブデザイン（スマホ最適化）
- タッチ操作に最適化されたUI

## デザイン・UI方針

### シンプルで見やすいデザイン
- 情報を一目で把握できるレイアウト
- 直感的な操作フロー
- 必要最小限の情報表示

### タッチ操作に最適化
- ボタンサイズ: 最低44x44px（タップしやすいサイズ）
- 十分な余白とスペーシング
- スワイプやタップなどのジェスチャー対応

### 緊急時でも使いやすいUI
- 大きく読みやすいフォント
- コントラストの高い色使い
- シンプルな操作フロー（タップ数を最小限に）
- 重要な情報（賞味期限警告など）を目立たせる

## 開発ガイドライン

### コーディング規約
- セマンティックなHTML要素の使用
- BEM記法によるCSS命名規則（推奨）
- ES6+の記法を使用
- コメントは必要最小限に、コードの意図を明確に

### パフォーマンス
- 軽量で高速な動作
- 不要なライブラリの使用を避ける
- 画像の最適化

### アクセシビリティ
- 適切なARIA属性の使用
- キーボード操作への対応
- スクリーンリーダー対応
